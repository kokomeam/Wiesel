01

Purpose & Goals

WiseSel is "Cursor for courses." AI agents live inside the creation workflow (generate lessons, slides, knowledge checks, exercises, analytics, marketing), every learner gets an AI tutor, and — later — skill agents turn a finished course into an applied assistant (a sales course becomes a sales coach; a coding course a debugging tutor). That AI layer is the niche and the moat.

The studio itself is table-stakes — and that is the point

Course structure, lessons, slides, quizzes, and exercises are not the differentiator. Every competitor already has a competent one. The studio's job is to be instantly familiar and frictionless to anyone who has built an online course before — a faithful replica of the category's conventions — so the AI layer is the only thing that feels new.
Be conventional where we don't differentiate; radical where we do.

The mental-model error this PRD corrects

✗ What the current build models — an academic LMS

Canvas / Blackboard / Moodle. Quizzes with time limits, attempt caps, passing scores, difficulty tags; homework with points and due dates. A senior reviewer's reaction: "this looks like software for a university registrar, not a Udemy creator." The polish is real; the category is wrong — which is exactly why it doesn't read as production-level.

✓ What it should model — a creator-economy course

Udemy / Kajabi / Teachable. Self-paced, low-stakes, "pace yourself," "check your understanding." Learners are adults who bought a course, not students being graded. Nothing gates their progress; nothing locks. Knowledge is checked, never scored against them.

Goals

Familiarity

A creator who has used Udemy can build a module without a tutorial. Conventional curriculum tree, conventional block types, a guided Plan → Create → Publish flow.

Low-stakes by default

No score, timer, attempt cap, or due date ever gates a learner. Knowledge checks confirm understanding; exercises invite practice.

AI-ready seams

Every artifact exposes a clean hook for the AI layer and the analytics engine — the differentiation plugs in without reshaping the studio.

Audiences

Creators

Educators, SMEs, trainers, and competition coaches (USACO / FBLA). They bring expertise; the studio + AI turn it into a sellable, self-paced course. Power users (coaches) occasionally want stricter quiz controls — supported, but never the default.

Learners

Adults who bought a course to learn at their own pace. They want momentum and clarity, not grades. Their correctness on knowledge checks becomes a private comprehension signal for the creator — never a public or self-facing grade.

Scope

In scope — the basic studio

  • Course → module → lesson → block structure
  • Slide decks (existing Google-Slides-like editor)
  • Knowledge-check quizzes (low-stakes)
  • Practice Exercise (renamed homework, self-paced)
  • Guided creation flow / checklist (Plan → Create → Publish)
  • The hooks where AI & analytics attach

Out of scope — design so it slots in

  • future Student AI tutor (deep behavior)
  • future Creator AI agents' deep behavior
  • future Skill / coaching agents
  • future Live calls
  • future Marketplace, payments, Stripe
  • future Learner runtime & manual grading engine

Competitive Analysis

The reference set is the creator-economy course platforms — with Udemy as the primary UX anchor (its Plan / Create / Publish flow, curriculum builder, and low-stakes Practice Test / Assignment). The academic LMSs are included as an anti-reference: the category we are deliberately moving away from.

No live captures were taken in this session; competitor behaviors below are inferred from well-known product patterns and the Udemy course-creation flow described in the brief. Treat specifics as reasoned inference, not measured fact.

Where each platform sits

PlatformCreation flowAssessment model Gates progress?AI assistance
Udemy anchor Guided Plan → Create → Publish with a persistent checklist of curriculum, landing, pricing items Low-stakes Practice Test + Assignment (instructions + optional solution) No — self-paced Minimal (some AI assist for outlines/captions)
Teachable Curriculum builder (sections → lectures), simpler linear setup Lightweight quizzes; graded quizzes optional add-on No (optional drip/sequencing) Limited
Kajabi Product → outline builder, marketing-first Assessments & surveys, low-stakes No Creator AI (copy, outlines)
Skillshare / Thinkific Project-based (Skillshare) / structured builder (Thinkific) Class projects / quizzes, ungraded No Limited
Canvas / Moodle anti-ref Course-shell + gradebook centric Timed, attempt-capped, passing-score, weighted, due-dated Yes — scores & prerequisites lock content Mostly none

Takeaways

Borrow
  • Udemy's Plan → Create → Publish stepper + persistent side checklist
  • Low-stakes Practice Test framing → our Knowledge Check
  • Assignment with optional solution → our Practice Exercise
  • Never gate the learner; self-paced is the default everywhere
Simplify
  • One clean block menu, not a dense settings panel per artifact
  • Knowledge-check defaults to just questions + explanations — power settings hidden
  • Exercises drop points / due dates entirely from the default surface
Defer / differentiate
  • The AI layer is where we go beyond all of them — agents in the flow, a learner tutor, future skill agents
  • Comprehension analytics ("find & fix weak modules") — a creator superpower none of these foreground
The one-line competitive read: we win nothing by reinventing the curriculum builder, and we lose credibility by importing the LMS gradebook. Match the creator-economy convention exactly, then let the AI layer be the only surprise.

Product Approach

Keep the studio's structure conventional and re-skin the two assessment artifacts from "graded academic instrument" to "low-stakes creator-economy artifact," while preserving the data substrate the AI & analytics layers consume.

Approaches considered for the academic fields

ApproachTrade-offVerdict
A — Hard-delete from schemaPurest model, but discards data a future AI grader could use; hardest to walk backRejected
B — Demote behind "Advanced," keep substrateDefault surface is low-stakes; power users (coaches) retain controls; nothing lost; never gatesChosen ✓
C — Leave as-is, relabel onlyCheapest, but the academic feel survives — fails the reframeRejected

The guided creation flow

A familiar, Udemy-style progression with a persistent side checklist. The stepper is a guide, not a gate — a creator can move freely between phases. Publishing requires a small set of minimums; nothing here ever locks a learner.

Publish minimums (title + ≥1 lesson with content) gate only the Publish action — they never gate or lock a learner's progression through the course.

Key business rules — the invariants

IDInvariant
INV-1Nothing a learner does gates progression. No score, timer, attempt cap, or due date locks any lesson. The only eligibility predicate for "can the learner open this lesson" is true.
INV-2Knowledge checks are soft self-feedback. The learner sees immediate right/wrong + a short explanation, plus a soft "X of Y correct." Never a course grade, gradebook, or pass/fail.
INV-3Advanced quiz settings exist but are off by default and inert for gating. Even if a creator sets a passing score or time limit, it is metadata for power users — it still never gates progression (INV-1 wins).
INV-4Practice Exercise is fully optional & low-stakes. Rubric, solution key, and deliverable are each optional. No points; no due dates. The deliverable serves feedback (AI/creator) or self-check — not a grade.
INV-5Correctness is captured as analytics, not grading. Knowledge-check correctness feeds the creator's comprehension/engagement signal and the future "find & fix weak modules" AI — never a learner-facing gradebook or ranking.
INV-6AI & analytics attach at seams only. The studio exposes generate/improve hooks per artifact and a correctness-capture hook; their deep behavior is out of scope here and must not reshape the studio.

Conventional vs. radical — where each lives

Table-stakes — be conventional

  • Curriculum tree (course → module → lesson → block)
  • Block types & slide editor
  • Plan → Create → Publish flow + checklist
  • Low-stakes knowledge checks & exercises

Differentiation — be radical

  • AI agents in the creation workflow (generate / "Magic Wand" edits)
  • AI tutor for every learner future
  • Comprehension analytics → fix-weak-modules AI
  • Skill / coaching agents future

B1 — Description
  • Per-question feedback is immediate: right/wrong marker + the creator's explanation (INV-2).
  • Soft tally: a non-graded "X of Y correct" for self-orientation only — no percentage grade, no pass/fail, no gradebook entry. (Chosen over "explanations only" and "full % score.")
  • Continue is never disabled regardless of answers, timer, or any Advanced setting (INV-1 / INV-3).
  • Analytics hook (INV-5): the response's correctness is written to the comprehension signal (per question / lesson / module) — a private creator analytics event, invisible to the learner as a score.
  • Surface-specific: learner-only; reuses the warm system but with the learner accent (sky→teal) per the design system.

User Experience (Acceptance)

Core-flow acceptance from the user's perspective, based on visible prototype actions. Each story passes when the expected result holds.

US-1Creator builds a low-stakes knowledge check
User need
Confirm learners understood a concept — without it feeling like a graded test.
Flow
  1. Open a lesson → + Add block → Knowledge Check
  2. Add a multiple-choice question + an explanation
  3. Glance at the editor; optionally expand Advanced (optional)
Expected result
The default editor shows only questions, answers, and explanations — no timer, score, difficulty, or points. Those appear only inside the collapsed Advanced disclosure, off by default, labeled "never gates."
US-2Creator builds a self-paced practice exercise
User need
Give learners something to practice, with optional guidance — not a graded assignment.
Flow
  1. + Add block → Practice Exercise
  2. Write instructions; optionally add a rubric and a solution key
  3. Optionally enable a deliverable (Text / File / Link / None)
Expected result
There is no points field and no due-date field anywhere. Rubric, solution key, and deliverable are each optional and labeled as feedback / self-check guidance — not grading.
US-3Learner takes a knowledge check and is never blocked
User need
Check my understanding and keep moving at my own pace.
Flow
  1. Answer the knowledge-check questions
  2. Read the immediate right/wrong + explanation
  3. Press Continue →
Expected result
Each answer shows immediate feedback + explanation and a soft "X of Y correct" — no percentage grade or pass/fail. Continue is always enabled, whatever the answers.
US-4Creator follows the guided Plan → Create → Publish flow
User need
Know what to do next and when the course is ready to publish.
Flow
  1. Set a title, build the outline (Plan)
  2. Add lesson content (Create)
  3. Watch the side checklist and stepper update; reach Publish
Expected result
The checklist marks items done as the document fills; the progress bar advances. Publish enables once minimums (title + ≥1 content lesson) are met — and the flow never locks a learner's progression through the course.
US-5Creator reads comprehension as a signal, not a gradebook
User need
See where learners struggle so I can improve weak modules.
Flow
  1. Open Analytics
  2. Read the comprehension by module panel
  3. Note the AI insight on the weakest module
Expected result
The panel shows aggregate correctness per module framed as "where learners struggle" — no per-learner scores, rankings, or pass/fail. An AI-insight slot points at the weakest module (deep behavior future).

Scope note: acceptance covers the core studio flows only. Backend persistence, the real LLM, the AI grader/tutor, and marketplace/payments are out of scope and intentionally excluded from these criteria.
